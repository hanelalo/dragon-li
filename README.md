# Dragon-Li

## 提示词 (Prompts) 管理

应用中的核心提示词已外置到用户数据目录的 `~/.dragon-li/prompts` 文件夹中，以 Markdown 文件形式进行管理。这允许用户在不修改代码的情况下自定义 AI 的核心行为。

应用在首次启动时，会自动在 `~/.dragon-li/prompts` 下初始化以下提示词文件。系统会在每次调用大模型前实时读取这些文件，**修改文件内容后即刻生效**（无需重启应用）：

### 1. `title_generation.md`
- **作用**：对话标题生成模板。用于约束模型从用户的首条消息中提取 1-6 个字的极简标题。
- **改写规范**：
  - 不需要特定的变量占位符（代码会在 Prompt 末尾自动拼接用户文本）。
  - 建议保留强制要求语言与用户输入一致的指令，避免中文对话生成英文标题。
  - 保持约束指令，确保输出极简短，不要包含多余的思考过程、标点或换行。

### 2. `memory_extraction.md`
- **作用**：长效记忆自动提取模板。指示模型在后台异步分析用户的最新对话和历史上下文，提取关于用户的持久性事实、偏好、约束限制等。
- **改写规范**：
  - 不需要特定的变量占位符（代码会作为 `system` prompt 传入，后续自动附加对话历史）。
  - **必须**要求模型输出严格的 JSON 格式，且根节点为 `memories` 数组。
  - 提取记忆的 JSON 结构必须包含 `type_`（fact/preference/constraint/project/task）、`summary`、`evidence`、`tags` 和 `confidence` 字段，以确保系统数据管道能够正确解析。

### 3. `chat_system.md`
- **作用**：默认聊天系统提示词。定义 AI 作为智能助手的核心角色，以及如何使用检索到的长效记忆上下文来回答问题。
- **改写规范**：
  - **必须包含 `{current_time}` 变量**：用于在对话时动态注入系统当前时间。
  - **必须包含 `{memory_section}` 变量**：用于动态注入从数据库中召回的相关历史记忆内容。
  - 建议保留要求 AI 忽略无关记忆的指令，防止 AI 在回答中生硬地提及不相关的记忆。
