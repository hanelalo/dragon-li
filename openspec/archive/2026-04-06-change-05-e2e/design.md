# Design: change-05-p0-e2e-smoke-and-acceptance

## 验收策略

采用“手动主流程 + 可脚本化检查”组合：

- 手动验证 UI 与交互正确性。
- 脚本验证构建产物与关键命令可用性。

## 用例分层

- `L1 主链路`：可用 profile 下完整多轮会话，日志与消息落库成功。
- `L2 异常链路`：错误码与重试行为符合预期。
- `L3 安全边界`：越界路径/域名/能力请求被拒绝。

## 结果归档

- 在 `docs/release/` 下维护 P0 验收记录。
- 记录执行时间、执行人、环境、通过/失败、阻塞项。

## 退出标准

- 所有 L1 用例通过。
- L2/L3 无 P0 阻断级缺陷。
- 阻塞问题有明确修复计划与责任人（如有）。

## 风险与缓解

- 风险：仅人工验证遗漏边界情况。
- 缓解：将高风险路径沉淀为固定 checklist，后续持续复用。
