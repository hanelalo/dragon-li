# change-00-mvp-consistency-and-streaming-fixes

## 背景与问题

已归档 MVP 中存在若干“实现与文档口径不一致”问题，影响后续 P0 开发基线与验收可信度：

- `chat_send` 当前为响应完成后批量回放事件，不是真流式。
- 配置模块设计文档写“文件监听”，实现为“显式检查外部变更”。
- 软删除策略文案存在歧义（“自动软删除”）。
- 错误码文档包含 `DB_BUSY`，实现未稳定产出该错误码。

## 目标

- 修复流式链路，确保事件边到边发送。
- 统一配置变更检测口径（实现与文档一致）。
- 统一软删除策略表述，消除歧义。
- 落地 `DB_BUSY` 错误码与重试行为。

## 范围

- Rust 聊天链路流式传输与事件发射改造。
- 配置变更检测相关文档修订（与当前实现一致）。
- 软删除策略文档统一修订。
- SQLite busy/locked 错误映射与重试落地。

## 非目标

- 不处理“缺聊天页/配置页”的产品能力问题（属于后续 P0 change）。
- 不在本 change 内补完 Rust <-> Python 完整 IPC 协议（属于后续 P0 change）。

## 依赖与顺序

- 无新增前置依赖。
- 本 change 优先级高于 `change-01` 到 `change-05`，先完成再推进后续 change。

## 验收标准

- 聊天回复可实时流式显示（非批量回放）。
- 配置变更文档与实现口径一致。
- 软删除文档表述统一为“用户触发删除时默认软删除”。
- `DB_BUSY` 可被稳定映射并触发既定重试策略。
