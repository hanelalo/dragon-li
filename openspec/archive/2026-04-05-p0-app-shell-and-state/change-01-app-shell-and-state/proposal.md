# change-01-app-shell-and-state

## 背景与问题

当前桌面前端仅有单页 `Memory Center` 验证界面，缺少面向产品化的应用壳层（导航、路由、全局状态）。后续聊天页与配置页无法在统一结构下接入，容易导致状态割裂和重复改造。

## 目标

- 建立 P0 阶段统一应用壳层。
- 建立页面路由：`聊天` / `配置` / `记忆`。
- 建立全局状态容器，承载跨页共享状态。

## 范围

- 新增前端路由与导航框架。
- 重构现有 `App.vue` 为壳层 + 子页面结构。
- 定义全局状态最小模型（会话、配置、运行时状态占位）。

## 非目标

- 不在本 change 内实现聊天业务逻辑。
- 不在本 change 内实现 Provider 配置 CRUD。

## 依赖与顺序

- 本 change 为串行第一步，无前置依赖。
- `change-02/03/04/05` 依赖本 change 的壳层与状态基础。

## 验收标准

- 应用可在三个页面入口间切换。
- 页面切换不导致全局状态丢失。
- 现有记忆中心页面可作为子页面继续访问。
