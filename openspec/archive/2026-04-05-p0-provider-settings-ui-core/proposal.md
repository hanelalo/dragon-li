# change-04-provider-settings-ui-core

## 背景与问题

配置能力已在宿主层实现，但缺少产品化配置页面。用户无法在 UI 中完成 profile 管理与连通性验证，导致可用性不足。

## 目标

- 建立 Provider 配置页面闭环。
- 支持 profile 的新增、编辑、删除、启用/禁用。
- 支持配置校验与连通性测试反馈。

## 范围

- 配置页列表与编辑表单。
- schema 校验错误提示。
- 外部配置变更检测与“确认应用”交互。
- 配置保存后与聊天页状态同步。

## 非目标

- 不在本 change 内扩展新 Provider 类型。
- 不改动现有安全边界策略。

## 依赖与顺序

- 前置依赖：`change-01`、`change-02`、`change-03`。
- 串行顺序固定在第 4 步执行，不并行。

## 验收标准

- 用户可在 UI 内完整维护 profile。
- 非法配置在前端有清晰报错，不会覆盖现有有效配置。
- 外部文件变更可被检测并按确认逻辑应用。
