# Change 09: App Layout Refactor Tasks

## 1. 移除全局左侧导航栏

- [x] 在 `src/layouts` 或 `src/App.vue` 中移除包含全局导航菜单（Chat、Config、Memory）的组件。
- [x] 调整 `App.vue` 或 `MainLayout.vue` 的 CSS，消除左侧留白。
- [x] 调整应用的默认路由，使其直接重定向或指向聊天主界面。

## 2. 改造会话列表区域

- [x] 修改 `ConversationList.vue` 或对应的左侧会话列表组件，将其底部区域固定，并添加一个操作栏。
- [x] 在该底部操作栏中加入一个明显的“设置”（齿轮）按钮。

## 3. 设计并实现设置入口跳转

- [x] 决定点击齿轮按钮后的交互形式（推荐方案：新窗口/全屏路由页面或带遮罩的抽屉）。
- [x] 若采用聚合设置页：新建一个 `SettingsView.vue`，作为原 `Config` 和 `Memory` 两个功能的父容器，通过内部的 Tab 切换两个页面。
- [x] 若采用简单菜单：点击齿轮弹出菜单（Popover/Dropdown），包含“API 配置”和“记忆管理”两个选项，点击后分别打开对应的独立组件。
- [x] 更新相关的 Vue Router 配置，适配新的跳转逻辑。

## 4. 验证与清理

- [x] 验证移除全局菜单后，会话列表与聊天区的宽度自适应是否正常。
- [x] 验证从会话列表底部点击设置后，能够正确进入配置与记忆管理界面。
- [x] 验证从配置和记忆管理界面能够方便地返回聊天主界面（例如“关闭”按钮或“返回”按钮）。
- [x] 移除废弃的导航组件和无用的 CSS 样式。

## 验收标准 (Acceptance Criteria)

- [x] 应用启动后，首页直接展示“左侧会话列表 + 右侧聊天区”的两栏布局。
- [x] 不存在最左侧的全局导航菜单。
- [x] 会话列表底部存在明显的设置入口（齿轮图标）。
- [x] 通过该设置入口，用户能够顺利找到并进入“配置”和“记忆”管理界面，并且功能均正常可用。
- [x] 设置/记忆界面有清晰的“返回”或“关闭”机制，能够回到聊天首页。
