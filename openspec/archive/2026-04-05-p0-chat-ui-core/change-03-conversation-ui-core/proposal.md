# change-03-conversation-ui-core

## 背景与问题

目前前端没有产品化聊天主界面，会话与消息链路未在 UI 层形成闭环。即使运行时可调用，也无法支撑真实用户使用场景。

## 目标

- 建立聊天主界面与会话管理界面。
- 打通“会话选择 -> 发消息 -> 流式回复 -> 历史回看 -> 失败重试”闭环。

## 范围

- 会话列表（创建、切换、归档/恢复、基础重命名）。
- 消息时间线（用户/助手消息、流式增量渲染）。
- 输入区（发送、禁用态、重试入口）。
- 请求错误可视化（错误码 + request_id）。

## 非目标

- 不实现 Provider 配置编辑能力（归 `change-04`）。
- 不实现跨设备同步。

## 依赖与顺序

- 前置依赖：`change-01`、`change-02`。
- 后续 `change-05` 将基于本 change 进行端到端验收。

## 验收标准

- 用户可创建多个会话并自由切换。
- 消息可流式显示并落库后可重载。
- 失败请求可在 UI 明确看到错误并触发重试。
