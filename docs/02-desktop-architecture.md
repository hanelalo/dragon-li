# 02. 桌面架构与技术选型

## 技术路线（当前共识）

- 桌面宿主：Tauri（Rust）
- 前端：Vue 3 + Vite
- Agent 引擎：Python
- 检索：首版关键词检索，接口可扩展

## 架构分层

- UI 层（Vue）：页面、状态管理、交互。
- 宿主层（Rust/Tauri）：窗口管理、IPC、进程生命周期、系统能力。
- Agent 层（Python）：LLM 编排、记忆策略、Provider 适配。
- Capability 层（统一抽象）：`native` / `skill` / `mcp` 能力模型与调用协议。
- 数据层（本地）：SQLite（对话/索引/日志）+ Markdown（记忆）。

## 关键原则

- 不以独立对外后端服务形态运行。
- 桌面应用内聚，面向本地使用场景。
- Agent 能力首版收敛：不开放外部工具调用，不访问工程目录。
- Skill 与 MCP 在 MVP 仅做预埋，不开放执行。
- 为后续扩展向量检索预留统一检索接口。

## Prompt 分层（固定顺序）

1. System 层：产品身份、语言风格、安全边界。
2. Runtime 层：当前模型、时间、会话元信息、运行约束。
3. Memory 层：短期摘要 + 长期记忆 Top-N（带来源 ID 与置信度）。
4. User 层：用户本轮输入 + 最近若干轮对话。

## Memory 注入规则（MVP）

- 每轮注入上限：5 条。
- 注入条目必须携带来源 ID，保证可解释性。
