# 10. 安全边界细则

本文定义 `狸花猫（Dragon-Li）` 在 MVP 阶段的本地安全边界与默认策略。

## 文件系统边界

- 仅允许读写：`~/.dragon-li/` 及其子目录。
- 禁止访问：工程目录、系统目录、用户其他目录。
- 导出文件例外：仅允许用户通过“保存对话框”显式选择导出路径。

## 网络边界

- 仅允许访问：`api_profiles.json` 中已配置的 Provider 域名。
- 默认拒绝其他外联请求。
- MVP 禁止网页抓取与任意 URL 访问。

## 执行边界

- 禁止 shell/脚本执行。
- 禁止动态加载外部插件。
- 禁止调用本地高风险系统命令。
- `skill` 与 `mcp` 在 MVP 不开放执行，仅允许配置与展示。

## 配置边界

- 仅加载：`~/.dragon-li/config/api_profiles.json`。
- 检测到外部配置变更后，需用户确认才应用。
- JSON 或 schema 校验失败时，不覆盖当前运行配置。

## 数据边界

- 对话主存储：SQLite（`~/.dragon-li/data/dragon_li.db`）。
- 记忆存储：Markdown（`~/.dragon-li/memory/`）。
- 删除策略：用户触发删除操作时，默认执行软删除（写入 `deleted_at`）；硬删除需二次确认。
- 关联删除策略：禁止使用数据库外键，所有级联删除/恢复必须在代码层显式执行并记录日志。

## 日志与隐私

- 日志禁止记录完整 `api_key`。
- 错误日志记录 `request_id`、`error_code`、必要上下文，不记录敏感正文。
- MVP 阶段不启用遥测上报（纯本地运行）。

## 权限交互

- 所有潜在越界动作都必须显式弹窗确认。
- 弹窗需说明：访问对象、执行原因、影响范围。
