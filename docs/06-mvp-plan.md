# 06. MVP 迭代计划

## 目标

在 2 周内完成可用 MVP：

- 可视化对话
- 多 Provider 配置与调用
- 记忆候选审核与 Markdown 持久化

## 迭代建议

### 第 1 周

- 搭建桌面应用骨架（Tauri + Vue）。
- 跑通会话创建、发送、流式回复。
- 完成 Provider 配置页与连通性测试（OpenAI、Anthropic）。

### 第 2 周

- 完成记忆候选提取与审核写入。
- 完成记忆中心（筛选、查看、编辑基础能力）。
- 接入 SQLite 数据层并启用 WAL。
- 补齐日志、错误处理、验收回归。

## 当前已定关键决策

- 产品以桌面 UI 为核心，不走 CLI 形态。
- 运行态数据统一放在 `~/.dragon-li/`。
- 不做旧目录迁移策略。
- API 配置使用明文 `api_profiles.json`。
- 检索首版为纯关键词，但保留可扩展性。
- 对话主存储改为 SQLite，记忆继续使用 Markdown。
- 数据库建模禁止外键，关联一致性由代码层保证。
- 用户触发删除操作时，默认执行软删除（写入 `deleted_at`）。
- 首版禁用外部工具调用，仅访问 `~/.dragon-li/`。
- Skill 与 MCP 仅做能力预埋，不开放执行。
- 发布策略先聚焦 macOS（MVP 阶段）。

## 发布策略（MVP）

- 首发平台：macOS。
- 架构优先级：Apple Silicon 优先，Intel 作为兼容目标。
- Windows/Linux：放在后续阶段推进，不纳入本期 MVP 交付范围。
- 自动更新：MVP 阶段关闭，采用手动下载安装包升级。
